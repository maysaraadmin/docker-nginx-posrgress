FIREFOX SSL/HSTS FIX INSTRUCTIONS
==================================

You're seeing "Secure Connection Failed" because Firefox has cached
HTTPS-only settings for localhost. Follow these steps EXACTLY:

METHOD 1: Clear Firefox HSTS Cache (RECOMMENDED)
------------------------------------------------
1. Type "about:config" in Firefox address bar and press Enter
2. Click "Accept the Risk and Continue" 
3. In the search box, type: "security.tls.insecure_fallback_hosts"
4. Double-click on "security.tls.insecure_fallback_hosts"
5. Type "localhost" and click the + button to add it
6. Click the checkmark to save
7. Close the about:config tab
8. Clear browser cache: Ctrl+Shift+Delete → Clear browsing data
9. Navigate to: http://localhost/ (NOT https://)

METHOD 2: Private Window (QUICK FIX)
-----------------------------------
1. Press Ctrl+Shift+P to open Private Window
2. Navigate to: http://localhost/
3. This bypasses all cached SSL settings

METHOD 3: Different Browser (ALTERNATIVE)
----------------------------------------
1. Try Chrome, Edge, or Safari
2. Navigate to: http://localhost/
3. Other browsers don't have this HSTS cache issue

METHOD 4: Disable Firefox SSL Temporarily
----------------------------------------
1. Type "about:config" in Firefox address bar
2. Search for: "security.tls.version.min"
3. Set value to: 0 (disables TLS temporarily)
4. Navigate to: http://localhost/
5. Remember to reset this back to 3 after accessing Moodle

WHY THIS HAPPENS:
------------------
Firefox stores HSTS (HTTP Strict Transport Security) settings
that force HTTPS for certain domains. Once it learns that localhost
should use HTTPS, it refuses to use HTTP even if the server is
configured for HTTP only.
